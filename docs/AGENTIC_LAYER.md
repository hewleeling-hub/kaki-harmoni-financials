# Agentic Layer

## Risk Classification

### Low — Auto-execute (no approval)
- Suggest expense category from vendor + description (keyword match)
- Compute and display chair occupancy % on report page
- Flag unsettled reimbursements > 7 days old
- Auto-advance chair status: Running → Resting → Free at computed timestamps

### Medium — Draft shown to user, one-tap to confirm
- Create reimbursement record when payer = personal or staff_card (shown as "We'll log this as owed to [name] — confirm?")
- Mark a chair as Free early (before rest timer ends — staff confirms)

### High — Explicit approval required
- Mark a reimbursement as settled (money has moved)
- Void / delete a sale record

### Critical — Human only, no agent action
- Delete a day's worth of session data
- Modify past expense amounts
- Any action touching financial totals retrospectively

## Named Tools (v1)
| Tool | Action | Risk |
|---|---|---|
| `suggest_expense_category` | keyword match → fills ai_category fields | Low |
| `advance_chair_status` | patch chairs.status at computed time | Low |
| `create_reimbursement` | insert reimbursements row after expense save | Medium |
| `settle_reimbursement` | set is_settled=true, settled_at=now() | High |
| `void_sale` | soft-delete sale + sale_items | High |

## Audit Log Fields
Every tool call appends to `audit_logs`: `id, created_at, actor_id, tool_name, target_table, target_id, before_state (jsonb), after_state (jsonb), risk_level, approved_by`.

## v1 vs Later
- **v1:** advance_chair_status + suggest_expense_category run automatically; create_reimbursement drafts on save
- **Later:** daily cashflow narrative generated by LLM; anomaly detection on low session count; AI receipt OCR populating expense form
